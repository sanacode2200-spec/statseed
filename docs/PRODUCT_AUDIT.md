# Statseed 実利用者目線プロダクト監査

調査日: 2026-06-14

## 結論

Statseed の強みは、初学者が日本語UIでデータ確認から検定、グラフ出力まで進める点にある。一方、実際の臨床・研究データへ使うには、操作の簡単さ以上に「誤解析を防ぐ」「除外や前提を説明できる」「同じ結果を再現できる」「投稿先で使える形式にする」ことが必要になる。

次の開発軸は、機能数を増やすことではなく、**入力データから論文・発表資料までの一貫した研究ワークフロー**にすることを推奨する。

## 今回の改善

- 箱ひげ図の標準表示を、分布形状ではなく個別値 + 箱ひげへ変更した。
- 2群では Welch t 検定または Mann-Whitney U 検定、3群以上では ANOVA + Tukey または Kruskal-Wallis + Dunn-Holm を選択できるようにした。
- グラフ上には比較線とp値を最大3組まで表示し、全比較結果・多重比較補正後p値・効果量は表で確認できるようにした。
- 「有意差あり／なし」という断定表示を避け、p値の閾値との関係、効果量、分布を併読する注意を追加した。
- PNG / SVG / PDF 出力にも同じ比較注釈を反映した。

## 設計原則

1. **解析を選ばせる前に、データの役割と研究デザインを確認する**
2. **p値だけでなく、個別値・効果量・信頼区間・欠損数を並べる**
3. **プレビューと出力を一致させ、出力後の手作業を減らす**
4. **分析条件・除外・ソフトウェア版を保存し、再現可能にする**
5. **色だけに依存せず、白黒印刷や色覚多様性でも意味を保つ**
6. **医療・研究データは保存しないことを標準にし、保存時は明示する**

## P0: 先に直すべき問題

### 1. 変数の役割をユーザーが確定できない

`backend/services/upload.py` は、数値化できる列を原則すべて連続変数と判定する。そのため患者ID、施設コード、0/1フラグ、順序尺度が連続変数として解析候補に入り、意味のない平均値・相関・箱ひげ図を作れる。

**改善案**

- 読み込み直後に「ID / 連続 / 順序 / カテゴリ / 日付 / 除外」の役割確認画面を置く。
- `id`, `患者id`, `症例番号`, 一意値率が高い整数列をID候補として警告する。
- 0/1列や少数の整数値はカテゴリ・順序尺度候補として提示する。
- 解析画面では役割に合わない変数を選択肢から除外する。

### 2. アップロードした全データがブラウザに永続保存される

`frontend/lib/dataStore.ts` と `frontend/contexts/DataContext.tsx` は、アップロード結果の全値を `localStorage` に保存する。共有PCや実患者データでは、画面を閉じても値が残る。医療情報は要配慮個人情報を含み得るため、便利さより明示的な安全設計を優先すべきである。

**改善案**

- 標準はメモリ内のみとし、タブを閉じたら削除する。
- 永続保存は利用者が明示的に選択した場合だけ許可する。
- 氏名、ID、生年月日、住所等の候補列を検出し、アップロード前後に警告する。
- 「この端末に保存中」「データを削除」を常時見える位置に表示する。
- 利用目的、送信先、保持期間、削除方法を画面内で説明する。

### 3. 欠損による解析対象の除外が見えない

記述統計では欠損数を表示するが、群比較・相関・グラフでは、どの行が何件除外されたかを利用者が追いにくい。解析ごとに対象数が変わると、結果の解釈と論文記載を誤る。

**改善案**

- 各解析結果に「元の行数 / 使用数 / 除外数 / 除外理由」を必ず表示する。
- 対応あり解析ではペア単位の除外を明示する。
- 完全ケース解析などの欠損処理方針を結果・方法文へ出力する。

### 4. 検定選択ガイドが単純な正規性判定へ誘導する

Shapiro-Wilk のp値や `n >= 30` だけで検定を選ぶと、研究目的、外れ値、群ごとの分布、分散、対応関係を見落とす。今回、画面上の単純な説明と古いDunn未実装表記は修正したが、ガイド全体の再設計が必要である。

**改善案**

- 最初に「平均値の差」「分布・順位の差」「対応あり」「調整済み効果」などの推定対象を聞く。
- 正規性検定を合否判定にせず、個別値プロット、Q-Q plot、外れ値、分散を並べる。
- 推奨理由と、結果が不安定になる条件をセットで表示する。

## P1: 論文・スライドへ直結させる改善

### 5. プレビューと出力の描画エンジンが異なる

プレビューは Plotly、出力は Matplotlib であり、余白、フォント、注釈位置、軸範囲が完全には一致しない。利用者が最終出力をダウンロードするまで見た目を確定できない。

**改善案**

- 最終的には単一の描画仕様または共通レイアウトモデルへ寄せる。
- 出力前に実際のSVG/PNGをプレビューする「印刷プレビュー」を追加する。
- 1段組・2段組・16:9スライドなど、幅・高さ・DPIのプリセットを用意する。
- SVGコピー、PNGコピー、背景透過、余白、フォント、線幅を一画面で編集する。

### 6. 対応ありデータを表すグラフがない

前後比較や左右比較では、個人ごとの変化を線で結ぶことが研究デザインの理解に重要である。現在は対応あり検定がある一方、対応関係を表現するグラフがない。

**改善案**

- paired dot plot / before-after plot を追加する。
- 各対象を線で結び、差分の分布、中央値差または平均差と95%CIを併記する。
- 対応あり検定画面から、そのまま対応ありグラフを作成できる導線を置く。

### 7. 解析結果とグラフ作成が別作業になっている

同じ群・変数・検定を再度選び直すため、設定の食い違いが起きやすい。

**改善案**

- 「解析結果からグラフを作る」を主要導線にする。
- グラフ、検定、効果量、信頼区間、サンプル数、除外数を1つの分析オブジェクトとして保存する。
- 図注と方法文を自動生成し、利用者が編集してコピーできるようにする。

### 8. 推定値と信頼区間が不足する

2群検定の一部には効果量と信頼区間があるが、事後比較、Table 1、カテゴリ比較などでは、p値が主要結果になりやすい。

**改善案**

- 群間差、中央値差、比率差、オッズ比、相関係数など、目的に合う効果推定値と95%CIを優先表示する。
- グラフ上のp値注釈は任意表示とし、推定値 + CI の注釈も選べるようにする。
- 「統計的差」と「臨床的重要性」を分けて記述する。

### 9. Table 1 のp値が標準で自動計算される

`backend/services/stats/table1.py` は群があるとp値を自動計算する。特に無作為化試験のベースライン表では、群間のベースラインp値より記述統計や標準化差が適する場合がある。また連続変数は表示形式に関係なく Mann-Whitney / Kruskal-Wallis 固定である。

**改善案**

- p値は初期状態でオフにし、目的を説明した上で選択可能にする。
- standardized mean difference を追加する。
- 欠損数、単位、使用した検定を表注へ自動出力する。

### 10. 再現可能な解析記録がない

分析設定、実行日時、除外、ソフトウェア版、探索的・事前規定の区別を保存できない。

**改善案**

- 分析履歴と再実行可能な設定JSONを保存・出力する。
- 方法文へ、検定名、両側/片側、多重比較、欠損処理、ソフトウェア名・版を出力する。
- CSV結果、図、図注、方法文、設定を1つの解析パッケージとしてダウンロードする。

## P2: 対象ユーザーを広げる改善

- 回帰分析、共変量調整、反復測定、一般化線形モデルを追加する。
- 研究デザイン別に CONSORT / STROBE / STARD / PRISMA 等の確認項目を案内する。
- 色だけで群を区別せず、点形状、線種、直接ラベル、白黒確認を追加する。
- キーボード操作、コントラスト、スクリーンリーダー、ライトモードのリンク色を監査する。
- 主要ワークフローのブラウザE2Eテストを追加する。
- 内部名 `steel_dwass` は実装内容が Dunn + Holm なので、互換性を考慮しつつ名称を整理する。

## 推奨ロードマップ

### Phase 0: 誤解析とデータ保護

1. 変数役割の確認・上書き
2. ID・個人識別情報候補の警告
3. データ永続保存をオプトイン化
4. 全解析の使用数・除外数表示
5. 検定ガイドの再設計

### Phase 1: 研究ワークフロー

1. 解析結果からグラフ作成
2. 対応あり個別値プロット
3. 推定値・効果量・95%CIの一貫表示
4. 図注・方法文の自動生成
5. 最終出力プレビューと投稿・スライド用プリセット

### Phase 2: 再現性と分析範囲

1. 分析履歴・設定エクスポート
2. Table 1 の標準化差とp値オプション化
3. 回帰・共変量調整
4. 研究デザイン別レポーティング支援
5. アクセシビリティとE2Eテスト

## 調査根拠

### 国内・国内利用に直接関係する資料

- 個人情報保護委員会「医療・介護関係事業者における個人情報の適切な取扱いのためのガイダンス」  
  https://www.ppc.go.jp/personalinfo/legal/iryoukaigo_guidance/
- Okabe M, Ito K. Color Universal Design: 色だけで情報を伝えず、形・位置・線種等を併用し、美しさも維持する原則。  
  https://jfly.uni-koeln.de/color/

### 国際的な統計・報告・図表の資料

- American Statistical Association. Statement on Statistical Significance and P-Values. p値だけで結論を出さず、効果の大きさや重要性とは区別する。  
  https://www.amstat.org/asa/files/pdfs/p-valuestatement.pdf
- ICMJE Recommendations. 統計手法を再現可能な詳細さで記載し、p値だけに依存せず、効果量と不確実性を示す。図は縮小後も明瞭で自己説明的にする。  
  https://www.icmje.org/recommendations/browse/manuscript-preparation/preparing-for-submission.html
- Weissgerber TL, et al. Beyond Bar and Line Graphs. PLOS Biology. 小標本では要約だけでなく個別値を示し、対応ありデータでは対応関係を可視化する。  
  https://doi.org/10.1371/journal.pbio.1002128
- EQUATOR Network. 研究デザインに応じた報告ガイドラインを使い、透明で再利用可能な報告にする。  
  https://resources.equator-network.org/
- SAMPL Guidelines. 生物医学論文での統計解析と結果報告の基本指針。  
  https://resources.equator-network.org/reporting-guidelines/sampl/

## 判断

美しいグラフとは装飾が多いグラフではない。研究デザイン、個別値、推定値、不確実性が短時間で誤解なく読め、白黒印刷・縮小・スライド投影でも意味が保たれるグラフである。Statseed はこの基準を、個々の編集機能ではなく製品全体の既定値として実装するべきである。
